# 接口变更公告:对象存储直传 / 图片域名白名单 / 后台 TOTP 登录 / 双令牌与设备管理

> 发布:2026-07-20(Yiora-1,当日两次增补)  影响范围:客户端(Flutter)、管理后台、所有调用方
> 服务端已上线并通过全量回归(十套件含 [12.1]-[12.4] 断言)。
> **增补摘要(第 6 节)**:登录/注册响应新增 refreshToken/deviceId;新增 /auth/refresh 与设备管理接口;强制改密流程已按产品决策移除。

## 1. 新增:对象存储预签名直传(图片/APK 上传)

服务端不再经手文件字节,客户端三步完成上传:

```
① POST /api/v1/upload/presign        (用户 JWT)
   body: {"kind":"post","fileName":"a.png","size":2048}
   resp: {"uploadUrl":"http://<s3>/yiora/post/2026/07/xxx.png?X-Amz-...","fileUrl":"http://<s3>/yiora/post/2026/07/xxx.png","expireAt":...}
② PUT {uploadUrl}                     (body = 文件原始字节,10 分钟内有效,无需鉴权头)
③ 把 {fileUrl} 回填业务接口的图片字段(发帖 images[].url、资料 avatar/cover、软件 logo/images 等)
```

- kind 与限制:`avatar`(5MB) `cover`/`post`/`software`(10MB,图片:jpg/jpeg/png/webp/gif)`apk`(500MB,.apk)。
- 管理端同能力:`POST /admin/v1/upload/presign`(管理令牌),kind 另支持 `banner`/`deco`。
- dev 环境对象存储为 compose 内 MinIO(`localhost:9000`,控制台 9001);CORS 已验证支持浏览器直传。

## 2. 破坏性变更:业务图片强制域名白名单

以下字段的 URL **必须来自上一节直传返回的 fileUrl**(即我方存储前缀),外链一律返回 **40000**:

| 接口 | 字段 |
| --- | --- |
| PUT /api/v1/user/me | avatar、cover |
| POST /api/v1/posts、POST /api/v1/posts/:id/edit | images[].url |
| POST /api/v1/software | logo、images[] |
| POST /admin/v1/banners | image |
| POST /admin/v1/mall/decorations | preview |

- **不受影响**:软件下载地址 downloadUrl(网盘外链)、帖子分享链接 linkUrl、举报证据图(暂宽松)。
- 请求体结构不变,仅值的来源受约束;客户端把"手填/拼接图片 URL"的逻辑一律改为直传取 fileUrl。
- 错误提示形如"图片链接不合法,请通过上传接口获取"。

## 3. 管理后台登录协议升级(仅 /admin/v1,用户侧登录不变)

完整时序(前置能力:图形验证码、5 次错密锁 15 分钟、IP 白名单、首登强制改密):

```
GET  /admin/v1/captcha                  → {captchaId, image(svg dataURI)}
POST /admin/v1/login                    body: {username,password,captchaId,captchaCode}
  ├─ 未开二步验证 → {token, mustChangePwd, ...}
  └─ 已开二步验证 → {totpRequired:true, ticket}          (ticket 5 分钟一次性)
POST /admin/v1/login/totp               body: {ticket, code}  → {token, ...}
     code = 验证器 6 位动态口令(±30s 容差,同码防重放)或 10 位一次性恢复码
```

- 绑定/解绑:`POST /admin/v1/totp/setup → confirm {code}`(确认才生效,响应含 10 个恢复码,明文仅一次);`POST /admin/v1/totp/disable {code}`;超管可在账号管理 `POST /admin/v1/admins/:id {resetTotp:true}` 强制解绑。
- `mustChangePwd=true` 期间除 `/admin/v1/password` 外全部 40300。

## 5. Flutter 对接示例(与 app/ 现有 dio/ApiResponse 风格一致)

### 5.1 上传封装 `core/network/upload_api.dart`

```dart
import 'dart:io';

import 'package:dio/dio.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';

import 'api_client.dart';
import 'api_response.dart';

/// 预签名直传:presign → PUT 原始字节 → 返回可落库的 fileUrl。
/// kind: avatar / cover / post / software / apk(见公告第 1 节大小与类型限制)
class UploadApi {
  UploadApi(this._dio);

  final Dio _dio;

  Future<String> uploadFile(String kind, File file) async {
    final name = file.uri.pathSegments.last;
    final size = await file.length();

    final resp = await _dio.post<Map<String, dynamic>>(
      '/api/v1/upload/presign',
      data: {'kind': kind, 'fileName': name, 'size': size},
    );
    final presign = ApiResponse.fromJson<Map<String, dynamic>>(
      resp.data!,
      (d) => (d as Map).cast<String, dynamic>(),
    ).unwrap();

    // PUT 直传对象存储:绝对 URL 自带签名,不能带 Authorization,也不走 baseUrl
    await _dio.put<void>(
      presign['uploadUrl'] as String,
      data: file.openRead(),
      options: Options(
        headers: {
          Headers.contentLengthHeader: size,
          Headers.contentTypeHeader: 'application/octet-stream',
          'Authorization': null, // 覆盖鉴权拦截器,预签名 URL 不允许多余签名头
        },
      ),
    );
    return presign['fileUrl'] as String;
  }
}

final uploadApiProvider = Provider<UploadApi>(
  (ref) => UploadApi(ref.read(dioProvider)),
);
```

### 5.3 发帖多图(发布页,并发直传后组装 images)

```dart
Future<List<Map<String, Object>>> uploadPostImages(
  WidgetRef ref,
  List<File> files,
) async {
  final api = ref.read(uploadApiProvider);
  final urls = await Future.wait(files.map((f) => api.uploadFile('post', f)));
  return [
    for (final u in urls) {'url': u}, // 宽高可选:{'url': u, 'width': w, 'height': h}
  ];
}
```

### 5.4 失败处理建议

- PUT 返回 403:签名过期(>10 分钟)或时钟偏差 → 重新 presign 再传一次。
- 业务接口 40000 且 msg 含"链接不合法":说明传了外链/旧 URL,改走直传。
- APK 大文件:`kind: 'apk'`,dio 传 `file.openRead()` 流式上传避免整包进内存(示例 5.1 已是流式)。

## 6. 增补:双令牌与设备管理(2026-07-20 第二批)

### 6.1 登录/注册响应变更(向后兼容,新增字段)

`POST /api/v1/auth/login|register` 请求可选新增 `deviceName`(如 "Xiaomi 14");响应新增:

```json
{
  "token": "...",            // access token(有效期从 expireAt 读,不要写死)
  "refreshToken": "...",     // 30 天,轮换式一次性
  "refreshExpireAt": 1690000000,
  "deviceId": "bc4e858b7d8a17ee"   // 客户端持久化,刷新与设备管理都要带
}
```

- 三个值都要安全存储(flutter_secure_storage / TokenStorage);`deviceName` 建议传机型。
- **兼容性**:不传 deviceName 也能登录;旧 access token(无设备 claim)照常可用,不参与设备踢下线。

### 6.2 刷新协议(access 过期时静默续期)

```
POST /api/v1/auth/refresh        (免鉴权)
body: {"refreshToken":"<存的 rt>", "deviceId":"<存的 did>"}
resp: 同登录响应(新的 token + 新的 refreshToken)
```

- **轮换式**:每次刷新旧 refreshToken 立即作废,必须用响应里的新值覆盖本地存储;
  用旧 rt 重放会得到 40100(若因并发丢了新 rt,只能重新登录)。
- 收到 40100 的场景:rt 过期(30 天未活跃)、设备被踢、账号被封禁 → 清本地登录态回登录页。
- dio 拦截器建议:401/40100 时先尝试 refresh 一次并重放原请求,再失败才登出(参考管理后台 user store 实现思路)。

### 6.3 设备管理(个人中心「设备管理」页可直接对接)

```
GET    /api/v1/user/devices        → [{deviceId,name,ip,lastLoginAt,current}]  (current=本设备)
DELETE /api/v1/user/devices/:id    → 踢下线:该设备存量 access + refresh 即时失效
```

- 单账号默认最多 5 台设备(服务端配置),超出自动淘汰最久未活跃的一台(等效被踢)。
- 被踢设备的下一次请求会收到 40100「该设备已被下线」,客户端按登出处理。

### 6.4 管理后台配套变更

- 首登强制改密流程已按产品决策**移除**(login 响应不再含 mustChangePwd;改密仍可从头像下拉/安全设置主动进入)。
- 新增管理能力(Flutter 不受影响,运营同学使用):圈子管理、话题管理、帖子首页置顶/加精、
  忧珠手动发放/回收与全站流水查询、靓号库管理、协议管理(客户端经 `GET /api/v1/agreements/{user|privacy}` 拉取)、
  用户等级调整与头衔授予。

## 七、青少年模式与帖子分享口令(P 级收尾批次)

### 7.1 青少年模式开关(设置页「青少年模式」项)

```
GET /api/v1/user/settings          (JWT)
resp.data: {"teenMode": false}

PUT /api/v1/user/settings          (JWT)
body: {"teenMode": true}            // 可扩展设置对象,后续新设置项会加进同一接口
resp: code 0
```

- 开启后**消费类接口全部拒绝**并返回 `40300「青少年模式下无法使用该功能」`:
  付费帖解锁 `POST /posts/:id/unlock`、抽奖 `POST /lottery/draw`、
  装扮兑换 `POST /mall/decorations/:id/exchange`、靓号兑换 `POST /mall/prettynos/:id/exchange`。
- 客户端配套:开启时隐藏/置灰上述入口,并对 40300 做统一 toast 兜底;
  浏览、发帖、评论、私信不受影响(内容池与时长限制二期落地)。
- 产品口径:开关即时生效,当前无密码保护(二期再加监护密码)。

### 7.2 帖子分享口令

```
POST /api/v1/posts/:id/share       (JWT)
resp.data: {
  "code": "YRSG7ZCGFY",                       // YR + 8 位,去易混淆字符(无 0O1IL)
  "text": "【帖子标题】复制这段话打开 Yiora,输入口令即可查看:YRSG7ZCGFY"
}

GET /api/v1/share/{code}           (免鉴权)
resp.data: {"postId":11,"title":"...","summary":"前 60 字...","author":"昵称","authorId":7}
失败: 40400 口令无效/已过期(30 天)或帖子已下架
```

- 同一帖子 30 天内重复分享返回**同一口令**;`text` 可整段复制进剪贴板。
- 客户端建议流程:App 启动/回前台时读剪贴板,正则 `YR[2-9A-HJ-NP-Z]{8}` 命中后调解析接口,
  弹「XX 邀你看【标题】」确认卡片再跳帖子详情页(避免无提示直跳)。
- 解析接口免登录,落地页可做未登录预览+引导注册。

## 八、APK 大文件分片上传(断点续传,发布者)

单次 PUT 直传(第一节)保留用于图片;**APK 改走 S3 Multipart 分片**,上限放宽到 2GB,弱网中断只补缺失分片。

```
POST /api/v1/upload/multipart/init        {kind:"apk", fileName:"xx.apk", size:字节数}
  → {uploadId, key, partSize:8388608, urls:[{partNumber,url}...], expireAt}
     // 一次返回全部分片 URL(10 分钟有效);按 partSize 切文件,分片 n 对应 urls[partNumber=n]

各分片:HTTP PUT urls[n].url,body 为该分片原始字节;记录响应头 ETag(去引号)与 partNumber

POST /api/v1/upload/multipart/complete    {uploadId, key, parts:[{partNumber, etag}...]}
  → {fileUrl}                              // 合并完成,fileUrl 回填软件发布表单

GET  /api/v1/upload/multipart/parts?uploadId=&key=
  → {parts:[已完成分片+etag], urls:[缺口分片的新签 URL]}   // 断点续传:App 重启后只补缺口

POST /api/v1/upload/multipart/abort       {uploadId, key}    // 用户取消,清云端碎片
```

- Flutter 建议:并发 3~4 路 PUT(dio),本地持久化 `uploadId/key/etag 位图`;恢复时先调 parts 比对;
  URL 过期(40x)也走 parts 补签。任务与归属票据服务端保留 24 小时,过期需重新 init。
- 越权保护:uploadId 归属校验,非本人操作返回 40300;任务不存在/过期返回 40400。
