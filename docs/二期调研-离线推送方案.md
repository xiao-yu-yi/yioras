# 二期调研:离线推送(APNs/厂商通道)接入方案

> 2026-07 调研。现状:通知体系 = 站内信(notification 表)+ WS 在线实时推(wspush)。
> **App 进程被杀/离线时用户收不到任何提醒**(私信、点赞评论、审核结果、公告),留存关键短板。

## 一、基本事实(决定架构的约束)

- 国内安卓无 Google 服务,离线推送**只能走厂商系统级通道**(华为/荣耀/小米/OPPO/vivo/鸿蒙),没有别的路;
  iOS 走 APNs。自建长连接只覆盖"App 活着"的场景——我们的 WS 已经是这条路,自建方案无法解决离线。
- 厂商通道要**逐家申请**(企业开发者账号,OPPO/vivo 要求应用先上架,审核 1~3 工作日),
  且各家 API/限额/分类(重要/普通消息)政策不同。
- 聚合商(个推/极光/友盟/UniPush)价值 = 一个 SDK+一个服务端 API 帮你桥接全部厂商通道;
  免费版走共享通道(高峰延迟/限频),VIP 版(约 2000~5000 元/月)才有送达率保障。

## 二、方案对比

| 方案 | 成本 | 送达率 | 评估 |
| --- | --- | --- | --- |
| **A. 厂商通道直连 + APNs 直连(推荐)** | 免费(各厂商基础额度内) | 系统级,最高 | 服务端逐家对接(华为/小米/OPPO/vivo + APNs 约 5 家 API);无三方依赖、无月费、无数据外流;工作量集中在服务端一次性投入 |
| B. 聚合商免费版(个推/友盟) | 免费 | 共享通道,高峰不保 | 接入最快(1 SDK),但免费档离线送达率打折,且设备数据经第三方 |
| C. 聚合商 VIP(极光/个推) | 2000~5000 元/月起 | 与直连相当 | 用钱换工期;当前体量月费不划算 |
| D. 统一推送联盟(UPS) | - | - | 标准仍未全面开放对接,暂不可用 |

**结论:方案 A(直连)**,但**分两步走**:
- **第一步(现在可做,零外部依赖)**:服务端落地推送抽象层+设备 token 体系+离线判定,先接 **APNs**
  (协议公开,HTTP/2 + JWT,无需审核)与**华为 Push**(注册即用,不要求先上架);
- **第二步(App 上架后)**:补 OPPO/vivo/小米通道(它们要求上架审核);若届时人力紧张,可临时用
  聚合商免费版兜底过渡,抽象层不变。

## 三、接入设计(贴合现有架构)

```
notifylogic.Add / imlogic 发消息 → (现状)站内信落库 + wspush 在线推
                                  → (新增)对端不在线(ws 无连接)且有 push token
                                      → pusher.Send(token, 标题/摘要/深链) → APNs / 厂商 HTTP API
```

1. **设备 token 表** `push_token(user_id, device_id, platform, channel, token, updated_at)`:
   客户端每次启动上报(`POST /api/v1/user/push-token`);与既有设备体系(deviceId)对齐,
   踢设备/退出登录时同步清 token。
2. **抽象层** `internal/pkg/apppush`:`Send(ctx, token Channel, title, body, deeplink)`;
   驱动:apns(HTTP/2+JWT)、huawei(OAuth2+REST)、xiaomi/oppo/vivo(REST)逐个补;
   未配置的渠道静默跳过(与 imgscan/llm/meili 同一策略,本地/CI 零依赖)。
3. **离线判定**:ws 网关已有连接注册表 —— wspush.Push 返回"是否在线投递成功",失败(不在线)才走离线推送,
   避免在线时重复提醒。
4. **频控与聚合**:同会话私信 1 分钟内合并为一条("你有 N 条新消息");营销类公告走单独分类
   (厂商"普通消息"类目,受每日限额约束),互动/私信走"重要消息"类目。
5. **深链**:payload 带 `yiora://post/123` 式 scheme,客户端冷启动路由到对应页。

## 四、落地里程碑

- M1(1~1.5 天,可立即开做):push_token 表+上报接口+apppush 抽象层+**APNs 驱动**+
  离线判定接线(ws 在线探测)+冒烟(mock 驱动断言路由与频控)
- M2(0.5 天):华为驱动(开发者账号注册后)
- M3(上架后):小米/OPPO/vivo 驱动,逐家申请逐家上线;补消息分类与每日限额治理
- 客户端配合:Flutter 端集成各厂商 SDK 取 token 上报,处理通知点击深链(排期与 M1 并行)
