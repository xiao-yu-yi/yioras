## 1. 产品概述

### 1.1 产品定位

Yiora 是一款**移动端综合兴趣社区 App**，核心形态为「圈子化内容社区」，围绕社区延伸三大业务支柱：

1. **内容社区**：以圈子（兴趣部落）为组织单位的图文帖子流，支持话题、@好友、忧珠付费解锁、共创。
2. **软件资源库**：UGC 形式的应用/游戏发布与下载中心（社区软件库）。
3. **忧珠成长体系**：任务/签到产出忧珠（积分），消耗于装扮（头像框/气泡）、靓号、积分抽奖、付费帖解锁；配套私信 IM 与 AI 智能管家。

> 全站无现金交易：忧珠仅通过站内行为获取（不可充值），从源头规避支付与资金合规风险。

### 1.2 目标用户

- 玩机爱好者、软件/游戏资源分享者（对应圈子：玩机专区、绿色软件、源码仓库、GM 游戏、Steam 专区等）。
- 泛兴趣社交用户（闲聊、影视、活动线报等圈子）。

### 1.3 平台范围

- **移动端 App**：Android 优先（社区含 APK 下载场景），iOS 同步。
- **管理后台**：Web 端（运营、审核、配置）。
- **服务端**：统一 API + WebSocket 长连接服务。

## 3. 功能需求详述

### 3.1 账号与用户体系（P0）

**注册登录（仅邮箱）**
- 邮箱 + 密码注册；注册时发送邮箱验证码完成验证。
- 邮箱 + 密码登录；邮箱验证码找回/重置密码。
- 不做手机号、第三方（微信/QQ/Apple）、一键登录。
- 登录态：JWT Access Token（短效）+ Refresh Token（长效），支持多端互踢配置。
- 邮箱验证码防滥用：单邮箱/单 IP 频控、图形验证码兜底。

**用户资料**
- 头像、封面图、昵称、个性签名、性别、生日。
- 用户编号体系：注册即分配数字 UID；展示层使用「靓号 ID」（如 `N101215`），靓号可在靓号商城用忧珠兑换更换（见 3.9）。
- 我的二维码：生成个人名片二维码，扫码跳转主页。

**等级与成长**
- 经验值等级 `Lv.0 ~ Lv.MAX`（发帖、评论、签到、被赞等行为加经验，后台可配权重与每日上限）。
- 头衔/徽章体系：官方头衔（开发者、官方、圈主等）+ 活动徽章，展示在昵称旁。

**权益认证（无实名）**
- 达人/开发者等身份认证（人工审核，提交作品/资料佐证），通过后授予头衔徽章。
- 不做实名认证。

**账号安全与合规**
- 设备管理、注销账号（应用商店合规必备）、青少年模式开关（合规预留）。

### 3.2 首页（P0）

- **顶部导航**：三个一级 Tab「首页 / 应用 / 商城」+ 全局搜索入口（搜帖子/用户/圈子/话题/软件）。
- **公告 Banner**：轮播卡片，支持图文（如免责声明、活动公告），后台可配跳转（帖子/H5/圈子/无跳转）。
- **置顶精选**：运营置顶帖横条（如「Bug 反馈贴」），支持多条轮播。
- **推荐信息流（单列表）**：不做多 Tab，仅一条推荐流；排序 = 加权（时间衰减 + 互动热度 + 运营加权），一期可先按热度/时间简化。
- **帖子卡片**：头像（含头像框装扮）、昵称、等级徽章、头衔、发布时间、浏览量、标题（红色高亮支持，运营帖）、正文摘要（折叠）、图片（1/2/3 宫格 + `+N`）、话题标签、点赞/评论数。
- 下拉刷新、上拉分页、骨架屏、列表数据 SWR 式缓存（先展示缓存后台刷新，解决重复加载）。

### 3.3 帖子与互动（P0）

**帖子详情**
- 作者卡片（头像/昵称/时间/关注按钮/官方标签）、富文本正文（文字 + Emoji + 图片 + @用户 + 话题 + 外链卡片）、图片九宫格（点击大图预览/保存）。
- 附加卡片：链接卡（站外链接，需白名单/风险提示）、抖音卡（解析分享链接）。不做商品卡。
- 互动：点赞、收藏、评论（两级：评论 + 回复，支持 @，图片评论可二期）、分享（站内转发 + 生成分享图/口令）。
- 浏览量统计（去重规则：同用户同帖 24h 计 1 次；仅作帖子热度展示，不做个人曝光统计）。
- **忧珠付费解锁**：帖子可设忧珠价格，未解锁用户只见摘要，消耗忧珠后解锁全文，忧珠转入作者忧珠账户（平台抽成比例后台可配）。
- 共创者：一篇帖子可添加共创作者，共创双方主页均展示该帖。
- 举报、不感兴趣、（作者/管理员）删除、编辑。

**话题**
- `#话题#` 体系：发帖时可选/可创建话题；话题聚合页（按热度/时间聚合帖子）；话题搜索。

**关注关系**
- 关注/取关用户；粉丝列表/关注列表；关注流（二期可加「关注」Tab）。

### 3.4 圈子（P0）

- **发现圈子页**：推荐圈子列表，支持「最热 / 最新」排序；圈子卡片（图标、名称、一句话简介、置顶角标）。
- **圈子详情页**：封面、简介、成员数、帖子数、加入/退出按钮；圈内信息流（置顶帖 + 最新/最热 Tab）。
- **圈子体系**：发帖必须选择一个圈子（发布器中「圈子」为必选项）。
- **圈子管理**：圈主/管理员角色 —— 圈内置顶、加精、删帖、禁言成员；圈子创建一期仅后台开通，二期开放用户申请。
- 官方特殊圈子：官方公告（仅官方可发）、骗子举报（曝光治理专区，强化审核）。

### 3.5 发布系统（P0）

- **发布入口**：底部导航中央 `+`，弹出半屏动作面板，两个通道：
  1. **发动态**（帖子）
  2. **发软件**（社区软件库）

**3.5.1 发动态**
- 标题：≤30 字（计数器），选填但鼓励填写。
- 正文：纯文本 + Emoji，支持输入 `@` 唤起好友选择器。
- 图片：0–9 张，支持排序/删除；上传前压缩。
- 附加内容（二选一）：外部链接 / 抖音链接。
- 圈子：**必选**，唤起圈子选择器。
- 话题：可选，多选（≤5 个），支持搜索与新建。
- 共创者：可选，选择互关好友，对方确认后生效。
- 权限设置：公开 / 仅粉丝 / 仅自己（预留）。
- 忧珠付费解锁：开关 + 忧珠定价（平台设定价格区间）。
- 重置、取消（存草稿提示）、发布（进入审核流）。

**3.5.2 发软件**
- 软件 Logo（必传，方图裁剪）。
- 介绍图：**3–6 张强校验**（少于 3 张禁止提交），支持长截图。
- 软件名字（必填）、软件简介（≤1000 字，计数器）。
- 发布类型：应用 / 游戏（单选 Tab）。
- 软件分类：后台可配的二级分类（软件/游戏 → 细分类目）。
- 软件版本（如 `2.3.1`）、软件大小（如 `128MB`）、软件渠道（选择器：官方/第三方/自制等，后台可配）。
- APK 标签：多个自定义标签（如 免登录、去广告）。
- 下载链接：http/https 校验；支持网盘链接 + 提取码字段（二期支持直传 APK 到对象存储）。
- 确认发布 → 审核队列（新软件与版本更新均需审核）。

**审核流（双通道通用）**
- 机审（文本敏感词 + 图片鉴黄鉴暴恐，云内容安全 API）→ 命中转人审 → 通过发布 / 驳回（附原因，可修改重提）。
- 审核状态：待审核/已发布/已驳回/已下架，作者可在「我的发布」查看。

### 3.6 应用中心 / 社区软件库（P0）

- 首页顶部 Tab「应用」进入：Banner 推荐位（含运营精选软件合集）+ 分类筛选（应用/游戏 → 二级分类）+ 排序（最新/最热/下载最多）。
- 软件详情页：Logo、名称、版本、大小、渠道、标签、介绍图横滑、简介、下载按钮（跳转下载链接/浏览器）、发布者信息、评论区、举报。
- 版本管理：同一软件可发布多版本，详情页展示历史版本列表。
- 下载统计：下载点击计数，用于热度排序。
- 安全提示：非官方渠道下载风险弹窗 + 免责声明（法务文案）。

### 3.7 消息与 IM（P0）

**消息主页**
- 单一私信会话列表（不做群聊）。
- 聚合通知入口（固定三项）：
  - 赞与收藏：谁赞/收藏了我的帖子与评论。
  - 评论和@：评论、回复、@ 我的消息。
  - 系统通知：审核结果、官方公告、账号通知。
- 官方 AI 管家会话（置顶，官方机器人标签）：新用户自动出现，支持 FAQ 问答、关键词回复（一期规则引擎，二期接大模型）。
- 底部合规提示：「聊天内容受平台实时监管，请文明交流」。

**私信**
- 互关免打扰限制策略（未互关每日可发条数受限，防骚扰，后台可配）。
- 消息类型：文本、表情、图片、帖子/软件分享卡片。不做红包。
- 已读/未读、撤回（2 分钟）、删除会话、拉黑举报。

**技术要求**
- WebSocket 原生长连接推送（连接鉴权、心跳、断线重连、离线消息拉取补偿），**不做轮询**。
- 未读数：会话级 + 全局角标；多端同步已读。
- 离线推送：厂商通道（华为/小米/OPPO/vivo/荣耀）+ APNs。

### 3.8 个人中心（P0）

**个人主页（自己/他人通用）**
- 封面图（可更换）、头像（含头像框装扮渲染）、昵称 + 等级徽章 + 靓号 ID、个性签名。
- 操作区：自己 → 编辑资料 / 更换封面 / 忧珠资产；他人 → 关注 / 私信。
- 数据栏（5 项）：关注数、粉丝数、获赞数、帖子数、忧珠（仅自己可见忧珠明细入口）。不做余额与曝光。
- 内容双 Tab：
  - 发布作品：我发布的帖子（含审核状态标记，仅自己可见状态）。
  - 我的足迹：浏览历史（仅自己可见，可清空）。

**侧边抽屉菜单**
- 我的设置（账号安全/通知开关/隐私/清缓存/关于/注销）。
- 兑换记录（装扮/靓号/抽奖的忧珠兑换流水）。
- 权益认证（达人/开发者认证）、头像框装扮、任务中心、靓号商城、更多（后台可配扩展位）。
- ~~我的二维码、气泡商城~~（v1.1 修订 6 裁剪）。

### 3.9 成长激励与忧珠商城（P1）

**任务中心**
- 每日任务（签到、浏览、点赞、评论、发帖、分享）+ 新手任务 + 成长任务，奖励忧珠/经验/装扮体验卡；后台全配置化。
- 连续签到阶梯奖励。

**忧珠（积分）体系**
- 获取：任务、签到、运营发放、付费帖被解锁分成；消耗：装扮兑换、靓号兑换、积分抽奖、解锁付费帖。
- 忧珠流水明细可查；忧珠不可充值、不可提现、不可转赠（防刷与合规基线，付费帖分成为唯一用户间流转场景）。

**积分抽奖**
- 转盘/宝箱抽奖，消耗忧珠；奖池：忧珠、装扮、靓号（运营配置概率与库存，概率公示合规）。

**装扮商城**

### 4.3 关键技术设计要点

1. **Feed 流**：一期「拉模式」（按圈子/全站索引表 + 多级缓存）；推荐单列表用离线计算的热度分（互动加权 + 时间衰减）定时刷新进 Redis ZSet。
2. **计数系统**：点赞/评论/浏览/粉丝等计数走 Redis 原子增减 + 定时/阈值回写 MySQL，防抖去重（浏览量 24h 去重用 Redis SETEX）。
3. **IM 消息模型**：会话（conversation）+ 消息（message）双表；序列号 seq 保证有序；在线走 WS 推送，离线走厂商推送 + 登录补拉；已读回执按会话 last_read_seq。
4. **忧珠账务安全**：忧珠账户 + 流水双表，所有变动（任务发放、抽奖、兑换、付费解锁分成）走事务 + 幂等键；每日对账任务核对余额与流水。
5. **付费内容（忧珠解锁）**：帖子内容分段存储（免费摘要/付费全文），服务端校验解锁记录后才下发付费段，防止客户端绕过。
6. **审核流水线**：发布 → 入库（状态=待审）→ MQ → 机审（文本/图片并行）→ 自动通过/转人审 → 状态机流转 → 通知作者；全渠道复用同一条管道。
7. **装扮渲染**：头像框/气泡为叠加素材（PNG/APNG/Lottie），客户端按 `decoration_id` 从 CDN 拉素材并缓存，佩戴信息随用户资料下发。
8. **API 规范**：REST + 统一响应包（code/msg/data/traceId）；接口版本 `/api/v1/`；签名 + 时间戳防重放；敏感接口（忧珠变动/兑换）二次校验 + 幂等。

## 7. 风险与注意事项

1. **合规风险（高）**：原型属「资源分享」社区，软件库存在版权与灰色资源风险。Yiora 必须建立软件上架审核白名单机制，禁止侵权/破解内容，否则存在下架与法律风险；免责声明不能替代平台审核义务。
2. **实名合规（需法务确认）**：本版不做实名认证，但境内上架时《互联网跟帖评论服务管理规定》等要求发帖评论用户真实身份核验；需在上架前确认目标市场与合规口径，必要时预留手机号核验开关（功能开关形式，不影响现有邮箱登录主流程）。
3. **IM 监管**：私信需按监管要求落实实时审核、拦截与留证能力。
4. **iOS 上架**：软件库（APK 下载）功能在 iOS 端需隐藏或改为「应用推荐」形态，做好双端功能差异开关；忧珠不可充值购买，不涉及 IAP。
5. **忧珠经济风险**：忧珠虽非现金，但抽奖概率需公示、防脚本刷取需风控；忧珠定价与产出投放需运营侧持续调平，防止通胀导致激励失效。
6. **性能演进**：一期单体架构需严格保持模块边界，IM 网关与业务 API 从第一天起分开部署，避免后期拆分成本过高。
