**量级估算**(Yiora 初期日均 1 万张 → 月 30 万张,需同时覆盖色情+暴恐+广告 3 场景):

- 腾讯 IMS:单次含全场景,按量 25 元/万 → **月约 75 元**;上量后 180 万张包 4000 元(22 元/万)
- 阿里云:postImageCheck 30 元/万(含社区场景标签)→ 月约 90 元
- 易盾/数美:按场景叠加,3 场景等效 45+ 元/万 → 月 135 元起,且需商务谈价

## 二、结论

**首选腾讯云 IMS**:单次调用出全场景标签(成本模型简单)、价格透明可自助开通、3000 张免费额度够完成
联调验证、异步检测原生带回调,与我们「直传后异步送审」的架构完全契合。
**备选阿里云**(postImageCheck 社区场景语义更贴合,若腾讯审核效果误杀率高可切换——接入层做成可替换驱动)。

## 三、接入设计(贴合现有代码)

```
用户直传 MinIO → 业务接口落库(帖/评/资料图,现状不变)
                    └─(新增)异步投递机审 → 云端回调/轮询
                          ├─ pass    → 无操作
                          ├─ review  → sensitivemodel.AddAudit 进现有人审队列(machine_detail 带标签/分值/命中图)
                          └─ block   → 直接下架(status=2)+ 站内信通知作者,记 audit 供申诉
```

1. **抽象层** `internal/pkg/imgscan`:`Scan(ctx, url) (verdict, labels, err)` 接口 + tencent/aliyun 两个驱动,
   `Config.ImgScan{Provider, SecretID, SecretKey, Region, BizType}` 选择;**未配置即跳过**(保持现状,本地开发/CI 不依赖外网)。
2. **送审时机**:帖子/评论创建与编辑成功后 goroutine 异步送审(不阻塞发布);头像/封面属高危位,
   用 profilePhotoCheck 类场景同步优先。MinIO 私有桶时给云端下发临时签名 GET URL。
3. **结果落地**:复用 audit_queue 人审工作台,后台审核详情页展示机审标签(machine_detail JSON 已预留);
   block 直接下架走既有 Takedown 通路,可人工翻案恢复。
4. **护栏**:云端超时/失败默认放行+记日志(降级为纯人审,不因外部依赖阻断业务);QPS 限流与重试(1 次)在驱动内做。

## 四、落地里程碑

- M1(1~2 天):imgscan 抽象+腾讯驱动+发帖/评论异步送审+回调路由+冒烟(mock 驱动)
- M2(0.5 天):后台审核详情展示机审标签;头像/封面同步审
- 上线前:开通正式账号,用免费 3000 张跑真实样本回归误杀率,再决定 block 阈值(初期建议只 review 不自动 block)
